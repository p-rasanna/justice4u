✅ Do's and Don'ts

Guidelines to maintain design system integrity and user trust.

✅

Use Semantic Colors

Always use semantic color tokens (--j4u-success, --j4u-danger) instead of hardcoded hex values for states and feedback.

❌

Don't Override Bootstrap Defaults

Avoid adding custom styles that conflict with Bootstrap. Use our CSS custom properties to customize Bootstrap components.

✅

Maintain Type Hierarchy

Use the established type scale. Don't create new font sizes or weights without adding them to the design system.

❌

Don't Skip Accessibility

Every interactive element must have proper focus states, ARIA labels, and keyboard navigation support.

✅

Use Motion Rules

Apply standardized motion classes (.j4u-motion-hover, .j4u-motion-success) instead of custom animations.

❌

Don't Use Generic Loaders

Always implement proper loading states with skeleton screens or progress indicators instead of generic spinners.

🎯 Accessibility Standards

Justice4U meets WCAG 2.1 AA standards for legal platform accessibility.

  • Color Contrast: All text meets 4.5:1 contrast ratio minimum
  • Focus Management: Visible focus indicators on all interactive elements
  • Keyboard Navigation: Full keyboard accessibility for all features
  • Screen Readers: Proper ARIA labels and semantic HTML
  • Reduced Motion: Respects user motion preferences
  • Touch Targets: Minimum 44px touch targets on mobile
